2025 Quine Explained

Date: December 29, 2024

This document analyzes a self-replicating JavaScript quine that displays "2025" using binary patterns and color manipulation. The code creates an animated display where each digit is represented by a 16-bit binary pattern. 


Original Quine Code:
--------------------

<body id=p bgcolor=x onload='setInterval(f=X/*Happy New Year Jo*/=>{for(i=0,o=`<pre>`,t++,b=61166,w=22,c="#fff",l=f+o;i<175;o+=++i%w?l[i].fontcolor([b,10408, 61102, 33442,b][i/w+6&7]&1<<i%w+t/5?c:t%w<i?t%w-i/w:c):`\n`);p.innerHTML=o},t=99)'>

Animation & Quine:
-----------------

setInterval(                                   // start animation 
  (f = (X) /*Happy New Year Jo*/ => {          // callback and comment (will be highlighted).
    for (                                      // loop 
      i = 0,                                   // iterator for pixel selection                 
      o = `<pre>`,                             // initial text as <pre> tag
      t++,                                     // increment the animation frame 
      b = 61166,                               // top line of '2025' which is the same as the bottom one. We save 5bytes.
      w = 22,                                  // width of the animation 
      c = "#fff",                              // default color (white) 
      l = f + o;                               // convert this function, all the code, in a string. It's our canvas for drawing. 
      i < 175;                                 // max character to print ( skip a bit for art) 
      o +=                                     // output html, made with all characters if the quine. 
        ++i % w                                // Check for line break, add a new line if needed ( see bellow )
          ? l[i].fontcolor(                    // get the character and set the color. Calling fontcolor create a 'font' element. It's a hack. 
              [                                // 2025, encoded in binary, stored as decimal, mirrored, line 1 to 5
              b,                               // 1 ⬛⬛⬛⬜⬛⬛⬛⬜⬛⬛⬛⬜⬛⬛⬛⬜  
              10408,                           // 2 ⬜⬜⬛⬜⬛⬜⬜⬜⬛⬜⬛⬜⬛⬜⬜⬜
              61102,                           // 3 ⬛⬛⬛⬜⬛⬛⬛⬜⬛⬜⬛⬜⬛⬛⬛⬜
              33442,                           // 4 ⬛⬜⬜⬜⬜⬜⬛⬜⬛⬜⬛⬜⬜⬜⬛⬜
              b][                              // 5 ⬛⬛⬛⬜⬛⬛⬛⬜⬛⬛⬛⬜⬛⬛⬛⬜
              (i / w + 6) & 7                  // select the correct row  
              ] &                              // 
                (1 << ((i % w) + t / 5))       // bit shifting for animation 
                ? c                            // -> main color ! 
                : t % w < i                    // Rain effect condition
                ? (t % w) - i / w              // -> rainbow character colors for rain effect 
                : c                            // else, use the main color for the comment highlight
            )                                  // 
          : `\n`                               // the new line mentionned earlier
    );                                         //  
    p.innerHTML = o;                           // Rewrite the page at each iteration 
  }),                                          // 
  (t = 99)                                     // Update each 99ms and.. instantiate our timer
);                                             // done !
